支付接口

  • `POST|GET /createOrder`： 创建支付订单。请求参数包含 `payId`、`type`、`price`、`sign`，可选 `param`、`notifyUrl`、`returnUrl`、`isHtml`。
  • `POST|GET /getOrder`： 按 `orderId` 查询订单详情，返回 `payId`、`orderId`、`payType`、`price`、`reallyPrice`、`payUrl`、`isAuto`、`state`、`timeOut`、`date`。
  • `POST|GET /checkOrder`： 查询订单支付结果。未支付或过期返回失败消息，支付成功返回同步跳转 URL。
  • `POST|GET /closeOrder`： 关闭未支付订单。请求参数包含 `orderId` 和简单签名 `md5(orderId + key)`。

监控接口

  • `POST|GET /getState`： 查询监控端状态，返回 `lastheart`、`lastpay`、`jkstate`。
  • `POST|GET /appHeart`： 监控端心跳上报，请求参数包含 `t` 和 `sign`。
  • `POST /appPush`： 监控端上报收款结果，请求参数包含 `type`、`amountCents`、`ts`、`nonce`、`eventId`、`sign`，并带防重放校验。
  • `POST|GET /closeEndOrder`： 清理超时未完成订单。

商户异步通知字段

  • `payId`： 商户订单号
  • `param`： 创建订单时传入的自定义参数
  • `type`： 支付方式
  • `price`： 订单金额
  • `reallyPrice`： 实际支付金额
  • `sign`： 回调签名，算法为 `md5(payId + param + type + price + reallyPrice + key)`
