VPay 商户对接文档

本文档面向商户系统开发者，只说明商户下单、查询、关闭订单、支付结果通知和签名验签。接口以当前系统域名为基础地址。

接入准备

基础地址

把示例中的 https://pay.example.com 替换为你的 VPay 站点域名。站点运行目录应指向 public/。

通讯密钥

在管理后台的支付基础配置中获取 通讯密钥。商户下单签名和回调验签都使用这个密钥。

推荐使用 application/x-www-form-urlencoded 提交参数。金额建议统一传两位小数字符串，例如 99.00；签名时必须使用和请求中完全一致的字符串。

商户支付流程

  1. 商户系统创建本地订单，生成唯一商户订单号 payId。
  2. 按 md5(payId + param + type + price + key) 生成下单签名，调用 /createOrder。
  3. 下单成功后，使用返回的 orderId 跳转到 /payPage/pay.html?orderId=平台订单号。
  4. 用户扫码付款后，VPay 会请求商户 notifyUrl，商户验签、更新订单并返回 success。
  5. 支付页检测到成功后调用 /checkOrder 获取同步跳转地址，并跳转到商户 returnUrl。

签名规则

场景 算法 说明
创建订单 md5(payId + param + type + price + key) param 未传时按空字符串参与签名；type 和 price 使用请求中的原始值拼接。
关闭订单 md5(orderId + key) 只用于关闭未支付的平台订单。
异步通知与同步跳转 md5(payId + param + type + price + reallyPrice + key) 商户验签时使用收到的字段字符串原文拼接，不要自行改金额格式。 
key = "你的通讯密钥"
payId = "M202604250001"
param = "user=10001"
type = "1"
price = "99.00"
sign = md5(payId + param + type + price + key)

创建订单

POST|GET /createOrder。推荐使用 POST。

参数 必填 说明
payId 是 商户订单号，必须唯一，最长 100 字符。
type 是 支付方式：1 微信，2 支付宝。
price 是 订单金额，必须大于 0 且小于 1000000。建议传两位小数。
sign 是 下单签名：md5(payId + param + type + price + key)。
param 否 商户自定义透传参数，会原样带回异步通知和同步跳转。
notifyUrl 否 本订单异步通知地址，最长 1000 字符；不传则使用后台默认异步回调地址。
returnUrl 否 本订单支付完成后的同步跳转地址，最长 1000 字符；不传则使用后台默认同步回调地址。
isHtml 否 传 1 时接口输出跳转脚本，直接跳到 VPay 支付页；默认返回 JSON。 
curl -X POST "https://pay.example.com/createOrder" \
  -d "payId=M202604250001" \
  -d "type=1" \
  -d "price=99.00" \
  -d "param=user=10001" \
  -d "notifyUrl=https://merchant.example.com/vpay/notify" \
  -d "returnUrl=https://merchant.example.com/order/result" \
  -d "sign=按签名规则生成的MD5"
{
  "code": 1,
  "msg": "成功",
  "data": {
    "payId": "M202604250001",
    "orderId": "202604251830001234",
    "payType": 1,
    "price": "99.00",
    "reallyPrice": "99.01",
    "payUrl": "weixin://...",
    "isAuto": 1,
    "state": 0,
    "timeOut": "15",
    "date": 1777113000
  }
}
reallyPrice 是实际应付金额，系统可能为了防止撞单微调金额。自定义收银台必须展示 reallyPrice，不要让用户按 price 付款。

当请求的支付方式暂不可用但存在其他可用支付方式时，接口仍返回 code=1，并返回 assignStatus=pending_choice。接入方仍按 orderId 跳转支付页，由用户在支付页选择其他支付方式。

查询订单

POST|GET /getOrder。按平台订单号查询支付页展示所需信息。

参数 必填 说明
orderId 是 /createOrder 返回的平台订单号。

订单状态 state：0 未支付，1 已支付，2 通知失败，-1 已过期，-2 已取消，-3 分配失败。

选择替代支付方式

POST|GET /selectOrderPayType。当订单处于 pending_choice 状态时，支付页调用该接口选择可用支付方式。

字段 必填 说明
orderId 是 /createOrder 返回的平台订单号。
type 是 用户选择的支付方式：1 微信，2 支付宝。

选择成功后，接口返回普通待支付订单数据，支付页会切换到二维码支付界面。

检查支付结果

POST|GET /checkOrder。支付页轮询使用；支付成功时返回商户同步跳转 URL。

参数 必填 说明
orderId 是 /createOrder 返回的平台订单号。 
{
  "code": 1,
  "msg": "成功",
  "data": "https://merchant.example.com/order/result?payId=M202604250001&param=user%3D10001&type=1&price=99.00&reallyPrice=99.01&sign=..."
}

未支付时返回 code=-1、msg=订单未支付；过期时返回 msg=订单已过期。

关闭订单

POST|GET /closeOrder。商户需要主动取消未支付订单时调用。

参数 必填 说明
orderId 是 /createOrder 返回的平台订单号。
sign 是 md5(orderId + key)。

只有未支付订单可以关闭。关闭成功后订单状态会变为过期状态。

异步通知

支付成功后，VPay 使用 HTTP GET 请求本订单的 notifyUrl。如果订单未传 notifyUrl，则使用后台默认异步回调地址。

字段 说明
payId 商户订单号。
param 创建订单时传入的自定义参数。
type 支付方式：1 微信，2 支付宝。
price 商户下单金额。
reallyPrice 用户实际支付金额。
sign 回调签名：md5(payId + param + type + price + reallyPrice + key)。 
GET https://merchant.example.com/vpay/notify?payId=M202604250001&param=user%3D10001&type=1&price=99.00&reallyPrice=99.01&sign=...
商户通知接口验签成功并完成业务处理后，响应正文必须返回小写 success。返回其他内容、请求超时或网络失败都会被视为通知失败。

同步跳转

支付页检测到订单已支付后，会跳转到 returnUrl。跳转 URL 会追加与异步通知相同的字段：payId、param、type、price、reallyPrice、sign。

同步跳转只适合给用户展示支付结果，商户订单最终状态应以异步通知验签后的处理结果为准。

PHP 示例

生成下单签名

<?php
$key = '你的通讯密钥';
$payId = 'M' . date('YmdHis');
$param = 'user=10001';
$type = '1';
$price = '99.00';

$sign = md5($payId . $param . $type . $price . $key);

异步通知验签

<?php
$key = '你的通讯密钥';

$payId = $_GET['payId'] ?? '';
$param = $_GET['param'] ?? '';
$type = $_GET['type'] ?? '';
$price = $_GET['price'] ?? '';
$reallyPrice = $_GET['reallyPrice'] ?? '';
$sign = $_GET['sign'] ?? '';

$expected = md5($payId . $param . $type . $price . $reallyPrice . $key);

if (!hash_equals($expected, $sign)) {
    http_response_code(400);
    echo 'fail';
    exit;
}

// 按 payId 查询商户订单，校验金额，幂等更新为已支付。

echo 'success';

接入注意事项

  • payId 必须唯一；重复下单会返回 商户订单号已存在。
  • 异步通知可能因网络问题重试或人工补单重复触发，商户侧必须按 payId 做幂等处理。
  • 商户通知地址必须公网可访问；系统会拦截指向内网或保留地址的通知 URL。
  • 使用 HTTPS 通知地址时，生产环境建议保持证书有效。后台关闭通知 SSL 校验只建议用于本地或自签名证书调试。
  • 自定义收银台可以直接使用 getOrder 返回的 payUrl，但必须展示 reallyPrice 并轮询 checkOrder。
